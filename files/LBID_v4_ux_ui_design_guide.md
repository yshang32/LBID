# LBID v4 — UX/UI Design Guide
## 點樣令用戶「一睇就知道點用」

> 配合 LBID_v4_complete_spec.md 使用
> 本文聚焦：視覺設計、資訊架構、互動模式，
> 令一個新用戶喺30秒內知道呢個平台係做咩、
> 同自己可以點用

---

# 1. 核心設計原則

## 1.1 「3 Question Test」每一個畫面

每個畫面必須喺3秒內答到用戶3條問題：

```
1. 我喺邊度? (Context)
   → 永遠有breadcrumb / 高亮navigation
   → 頁面有清晰title + sub-title

2. 而家發生緊咩? (Status)
   → 我嘅token、會員、未讀通知永遠可見
   → 進行中嘅交易狀態一目了然

3. 我可以做咩? (Action)
   → 最重要嘅CTA一定要係視覺焦點
   → 唔需要scroll就見到主要action button
```

## 1.2 「Action First, Filler Last」原則

```
✅ Dashboard最頂 = 用戶今日要做嘅事
   (可bid嘅SR / 等緊回覆嘅bid /
    進行中嘅交易)

❌ 唔好放: 抽象嘅統計圖、Marketing文案、
         "Welcome to LBID"嘅banner

→ Pro Tip: 用戶登入後第一個action
            應該喺3秒內可以發生
```

## 1.3 「Status永遠可見」原則

```
頂部固定bar永遠顯示:
  🪙 Token: 8 (本月剩餘額度)
  📊 信譽: ⭐47
  💎 會員: 月費 / 年費 / 試用中
  🔔 通知: 3

→ 用戶唔需要"搵"狀態，
   狀態主動進入視覺
```

## 3.3 為何用顏色 + 圖示 + 進度條?

```
研究顯示:
  - 純文字資訊: 認知處理時間 4-5秒
  - 圖示+顏色: 認知處理時間 0.8秒
  - 進度條/視覺化: 即時感知

LBID用戶大多係非技術出身嘅
中小企老闆/物流經理
→ 視覺資訊比文字更友善
→ 但唔可以過度卡通化
   (B2B需要專業感)
```

# 4. Navigation 互動模式

## 4.1 第一次登入 — Onboarding Tour

```
登入後第一次見到Dashboard:

Step 1: 半透明遮罩 + Spotlight 高亮
  → "呢度係你今日要做嘅事"
  → 箭咀指向"可接單SR" section

Step 2: 高亮Top Bar Token icon
  → "你嘅Token餘額永遠喺度顯示"

Step 3: 高亮Side Nav "Marketplace"
  → "更多接單機會喺呢度"

Step 4: 高亮Match Records
  → "進行中嘅交易記錄"

Step 5: "得喇!開始你嘅第一個bid →"
        → CTA直接帶用戶去第一個SR

→ 5步, 唔超過60秒
→ 可以"Skip Tour"
→ 完成後 user.onboarding_tour_completed=true
```

## 4.2 Empty States — 唔好淨係寫"無資料"

```
情境1: 仲未發過SR
┌────────────────────────────────┐
│ 📦 你仲未發過Shipment Request      │
│                                  │
│ 用平台搵forwarder幫你寄貨,        │
│ 5間公司會即時報價                  │
│                                  │
│ [發布第一個SR →]                   │
└────────────────────────────────┘

情境2: 仲未Bid過
┌────────────────────────────────┐
│ 🤝 你仲未Bid過任何Request          │
│                                  │
│ 而家有8個Open SR等緊你嘅報價      │
│ 你嘅Token餘額: 5 (足夠開始)        │
│                                  │
│ [睇睇有咩可以接單 →]              │
└────────────────────────────────┘

→ Empty state永遠帶action,
  唔好留用戶喺"我要做咩"嘅困境
```

## 4.3 Loading States — 唔好淨係轉spinner

```
Bid提交中:
  Spinner + "正在驗證你嘅Token..."
       ↓
       "建立Bid記錄..."
       ↓
       "通知Client..."
       ↓
       "✅ Bid提交成功!"
       
→ 用step-by-step進度，
   令3秒感覺似1秒
→ 用戶知道system"做緊嘢"，
   唔係hang機
```

# 5. 微文案 (Microcopy) — 講人話

## 5.1 動作按鈕

```
❌ 唔好: "Submit"
✅ 應該: "立即Bid -1 Token"
       (清楚做緊咩 + 成本)

❌ 唔好: "Update"
✅ 應該: "確認改動"

❌ 唔好: "Save"
✅ 應該: "儲存到我嘅範本"

❌ 唔好: "Confirm"
✅ 應該: "我確認提交 $1,500嘅報價"
       (重述行動內容，避免誤click)
```

## 5.2 提示/警告訊息

```
❌ 唔好: "Insufficient tokens"
✅ 應該: "Token不足喇! 你仲剩2個,
         呢個Priority Bid要2個。
         [購買更多Token]"

❌ 唔好: "Bid quota reached"
✅ 應該: "呢個SR已經滿名額喇 (5/5)
         你可以:
         [等下個SR] 或
         [建議Client加開名額]"

❌ 唔好: "Match completed"
✅ 應該: "🎉 配對成功!信譽分+5
         可以開始準備Quotation喇"
```

## 5.3 Status文字

```
❌ "Pending" → ✅ "等緊對方確認"
❌ "In Progress" → ✅ "交易進行中"
❌ "Expired" → ✅ "已過期(2日前)"
❌ "Active" → ✅ "進行中"
❌ "Inactive" → ✅ "已停用"
```

# 6. 顏色系統 (Color System)

## 6.1 主色調

```
Navy Blue (#1A2B4C) — 信任、專業 (主色)
Gold (#D4AF37) — 價值、特權 (CTA, VIP)
White (#FFFFFF) — 背景

語義色:
  Red (#DC2626) — 緊迫、警示 (即將截標)
  Green (#16A34A) — 成功、完成
  Orange (#EA580C) — 提醒、注意
  Gray (#6B7280) — 次要資訊
```

## 6.2 顏色運用規則

```
✅ Gold用嚮:
  - 主CTA按鈕("立即Bid")
  - VIP Badge
  - 信譽分數顯示
  - 重點數字(token餘額)

❌ 唔好濫用Gold:
  - 唔好用嚮普通文字
  - 唔好用作大面積背景
  - Gold係"指引視線"嘅工具,
    用得太多就失效

紅色"截標倒數":
  - 倒數時鐘字體閃動
  - 名額剩餘 ≤ 2 時, 進度條變紅
```

## 6.3 信譽分數視覺化

```
分數 0-19:   普通顯示，灰色
分數 20-49:  ⭐銅色, "活躍會員"標籤
分數 50-99:  ⭐銀色, "信譽認證"標籤
分數 100+:   ⭐金色, "金牌會員"標籤
分數 200+:   💎鑽石, "鑽石會員"標籤

→ 越多越想upgrade等級
→ 視覺差異要大, 一眼識別到
```

# 8. Match Record 詳情頁

## 8.1 5階段Progress Bar (升級視覺)

```
配對成立    報價已扣    資料解鎖    交易中    完成
   ●━━━━━━━●━━━━━━━●━━━━━○━━━━━○
   ✅          ✅          ✅          🔄

每階段都係:
  ✅ 已完成 (綠色實心)
  🔄 進行中 (脈動動畫, 金色)
  ⚪ 未到 (灰色空心)

Click任一階段 → 展開詳情側Panel:
  Stage 1: 顯示配對時間、winning bid細節
  Stage 2: token扣費記錄
  Stage 3: 對方完整聯絡資料 + WhatsApp按鈕
           (一click打開 https://wa.me/...)
  Stage 4: Quotation/AWB工具入口 + 備註
  Stage 5: 評分表格(1-5星 + 文字評語)
```

          享受Priority Bid特權"
  Day 7: "試用結束, 升級會員繼續使用"

→ 每個action都同會員嘅實際價值
  掛鉤, 唔係抽象marketing
