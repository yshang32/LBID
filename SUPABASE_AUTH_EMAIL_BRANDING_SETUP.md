# LBID Supabase Auth Email Branding Setup

## Problem

The signup confirmation email currently shows the default Supabase Auth sender and plain layout:

- Sender name: `Supabase Auth`
- Sender domain: `noreply@mail.app.supabase.io`
- No LBID logo
- No professional company layout

This email is generated by Supabase Auth, not by the LBID frontend. Changing the website code alone will not change this email.

## Desired Result

The confirmation email should look and feel like LBID:

- Sender name: `LBID`
- Subject: `Confirm your LBID account`
- LBID logo at the top
- Professional enterprise layout
- Clear confirmation CTA
- No visible Supabase branding where possible

## Step 1 - Update The Email Template

Go to Supabase Studio:

1. Open LBID project.
2. Go to `Authentication`.
3. Go to `Emails` or `Email Templates`.
4. Open `Confirm signup`.
5. Set subject:

```text
Confirm your LBID account
```

6. Paste the HTML from:

```text
supabase-auth-confirmation-email-lbid.html
```

7. Save changes.
8. Create a test signup and check the email.

The template uses this confirmation link variable:

```text
{{ .ConfirmationURL }}
```

Do not remove it. Supabase replaces it with the real verification link.

## Step 2 - Change Sender Name And Domain

To remove `Supabase Auth <noreply@mail.app.supabase.io>`, configure custom SMTP.

Recommended approach:

1. Use Resend or another SMTP provider.
2. Verify a sending domain, for example:

```text
lbid.com
```

3. Create a sender email:

```text
no-reply@lbid.com
```

4. In Supabase, go to Authentication email / SMTP settings.
5. Enable custom SMTP.
6. Set:

```text
Sender name: LBID
Sender email: no-reply@lbid.com
```

Without custom SMTP, Supabase may still show the default Supabase sender even if the email body is branded.

## Step 3 - Confirm Auth Redirect URLs

In Supabase Auth URL settings, confirm:

```text
Site URL: https://lbid.vercel.app
```

Recommended redirect URLs:

```text
https://lbid.vercel.app/**
http://127.0.0.1:*/**
http://localhost:*/**
```

This helps email verification return users to the correct LBID site.

If the confirmation link opens `localhost:3000`, the Supabase Site URL is still wrong or the email was generated before the redirect settings were updated. Create a new test signup after saving the settings.

Also set this Vercel environment variable:

```text
NEXT_PUBLIC_APP_URL=https://lbid.vercel.app
```

LBID's signup and password reset flows use this value to generate production-safe email redirects.

## Step 4 - Test Cases

Run these tests:

| Test | Expected Result |
| --- | --- |
| New signup | User receives LBID-branded confirmation email |
| Email logo | LBID logo loads in Gmail |
| Confirm button | Opens LBID and verifies account |
| Sender name | Shows LBID after SMTP is configured |
| Mobile Gmail | Email layout remains clean and readable |

## Important Notes

- Email clients require absolute image URLs, so the template uses:

```text
https://lbid.vercel.app/assets/lbid-web-logo-clean.png
```

- Do not use local paths like `/assets/logo.png` inside email templates.
- If the logo does not appear, check that the production URL is public and the asset is deployed.
- If the sender still says Supabase Auth, custom SMTP is not fully configured yet.
